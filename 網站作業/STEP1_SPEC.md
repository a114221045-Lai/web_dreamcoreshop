# 步驟 1：建立網站基本框架 - 需求規格

## 📋 概述
建立一個 Dreamcore 風格的網站基本框架，支援響應式設計、商品展示、購物車、聯絡表單及 AI 客服功能的基礎結構。

## 🎯 功能需求清單

### 1.1 首頁 (index.html)
- [ ] 網站頭部：logo、導航菜單（理念、商品、聯絡）
- [ ] Hero 區塊：視覺衝擊的標題與副標
- [ ] 理念介紹區塊：夢核藝術風格說明
- [ ] 商品展示區塊：25 件物品的網格展示
- [ ] 購物車按鈕：快速訪問購物車
- [ ] 聯絡表單：姓名、信箱、訊息欄位
- [ ] AI 客服聊天窗小工具：浮動按鈕與聊天界面
- [ ] 頁尾：公司資訊與連結

### 1.2 樣式設計 (styles.css)
- [ ] 色彩配置：
  - 主色 #1F2A44（暗藍，70-80%）
  - 次色 #7B7289（灰紫，15-25%）
  - 輔助色 #F2E2A2（淡黃，1-5%）
- [ ] 排版：
  - 使用系統字體棧（Noto Sans TC 等）
  - 合適的行高與字距
  - 響應式字體大小
- [ ] 動畫效果：
  - 平滑過渡（transition）
  - 懸停效果（hover）
  - 淡入淡出動畫
- [ ] 響應式布局：
  - 桌面版（1100px+）
  - 平板版（768px-1099px）
  - 手機版（<768px）

### 1.3 前端邏輯 (script.js)
- [ ] 商品數據管理：將 GOODs.md 的 25 件物品轉換為 JSON 數據
- [ ] DOM 操作：
  - 動態渲染商品列表
  - 展開/收起詳細描述
  - 購物車項目增減
- [ ] 事件監聽：
  - 聯絡表單提交（客戶端驗證）
  - 購物車按鈕點擊
  - AI 客服對話框開啟/關閉
- [ ] 本地存儲：購物車數據持久化（localStorage）
- [ ] API 調用：
  - 與後端 `/api/chat` 通信（AI 客服）
  - 處理響應與錯誤

### 1.4 購物車功能
- [ ] 購物車頁面 / 側邊欄（可選）
- [ ] 添加項目到購物車
- [ ] 移除購物車項目
- [ ] 編輯購物車項目數量
- [ ] 顯示小計與總價
- [ ] 結帳按鈕（可先為演示）

### 1.5 AI 客服 UI 基礎
- [ ] 聊天窗小工具（浮動按鈕）
- [ ] 對話框界面：
  - 消息顯示區域
  - 輸入框與送出按鈕
  - 加載指示器
- [ ] 樣式集成：Dreamcore 風格一致性

### 1.6 配置與構建
- [ ] `.env` 檔案模板（包含 PORT、API_KEY 等）
- [ ] `.gitignore` 檔案（排除 node_modules, .env 等）
- [ ] `vercel.json` 部署配置
- [ ] `README.md` 使用說明
- [ ] `package.json` 依賴管理

## ✅ 驗收標準

1. **功能完整性**
   - [ ] 所有上述功能模組均已實現
   - [ ] 無主要 JavaScript 錯誤

2. **設計一致性**
   - [ ] 色彩使用符合規範
   - [ ] 排版與間距統一
   - [ ] Dreamcore 風格明顯

3. **響應式測試**
   - [ ] 在手機、平板、桌面上顯示正確
   - [ ] 無水平滾動溢出

4. **性能**
   - [ ] 頁面加載時間 < 3 秒
   - [ ] 商品圖片優化（若有）

5. **可訪問性**
   - [ ] 語義化 HTML（heading, nav, main 等）
   - [ ] 適當的 alt 標籤
   - [ ] 鍵盤導航支持

## 📂 文件結構

```
網站作業/
├── index.html                 # 主頁面
├── styles.css                 # 全局樣式
├── script.js                  # 前端邏輯
├── .env                        # 環境變數（本地）

├── README.md                  # 專案文檔
├── STEP1_SPEC.md             # 本文件
├── package.json               # NPM 依賴
├── api-test.html             # API 測試頁面
├── GOODs.md                  # 商品數據
├── OR.md                     # OpenRouter API 文檔
├── SPEC.md                   # 總體規格
└── backend/
    ├── server.js             # Express 伺服器
    └── ORAPI.js              # OpenRouter 集成模組
```

## 🔄 開發流程

1. **確認需求** → 檢查本文檔，提出問題
2. **實現功能** → 逐步完成各功能模組
3. **本地測試** → 使用 `npm start` 或 `npm run dev` 測試
4. **驗收檢查** → 確認所有驗收標準已達成
5. **進入步驟 2** → 開始 AI 客服集成開發
